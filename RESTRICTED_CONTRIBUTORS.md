## How to Enforce Restrictions

### For Repository Maintainers

1. **GitHub Branch Protection Rules**:
   - Go to Settings > Branches
   - Add branch protection rules for `main` and other important branches
   - Require pull request reviews before merging
   - Review all PRs carefully before merging

2. **Code Review Process**:
   - Always check the contributor's username before merging
   - Reject any PRs from restricted contributors
   - Document any unauthorized contributions

3. **GitHub Settings**:
   - Consider limiting write access to specific collaborators only
   - Use branch protection to prevent direct pushes to main

### For Contributors

- If you see a pull request from a restricted contributor, please notify the repository owner
- Do not review or approve PRs from restricted contributors
- Report any unauthorized changes immediately
